# Core Refinement Challenges

## 1. Real Caller Variability

Real callers:
- interrupt
- ramble
- speak unclearly
- change direction
- provide information out of order
- become emotional
- contradict themselves
- forget information
- partially answer questions

The agent must remain:
- calm
- stable
- operationally focused
- non-robotic
- workflow-safe

The system must recover gracefully without:
- restarting intake
- losing operational context
- sounding confused
- escalating conversational chaos

---

## 2. Latency & Conversational Rhythm

Poor pacing destroys perceived quality.

Even strong prompts fail if:
- pauses feel unnatural
- interruption timing feels awkward
- response cadence feels synthetic
- confirmations feel repetitive
- turn-taking feels delayed

Latency quality depends on:
- Retell flow architecture
- model choice
- endpoint speed
- prompt structure
- orchestration design
- backend response performance

Voice refinement must include pacing refinement.

---

## 3. State Discipline

State discipline is foundational.

The agent must:
- not re-ask captured information
- not restart intake
- not loop after confirmation
- not invent workflow capabilities
- not promise unavailable scheduling
- not hallucinate dispatch commitments
- not skip required operational fields
- not violate backend workflow semantics

This is where the Gosonic workflow canon becomes foundational.

The conversational layer must remain subordinate to workflow truth.
